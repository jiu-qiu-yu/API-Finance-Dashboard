## Context

当前 API Finance Dashboard 是基于 PySide6 的桌面应用，已实现基础的巡检流程（浏览器抓取上游数据 → 计算盈利 → 展示结果）。现有界面为英文，布局较为朴素，缺乏面向 New API 站长的运营级可视化体验。数据模型（`SiteConfig`、`SiteResult`、`InspectionResult`）已包含余额、阈值、状态等字段，但 UI 层未充分利用。

现有模块结构：
- `ui/main_window.py` - 主窗口（英文，线性布局）
- `ui/status_list.py` - 状态列表（基础表格）
- `ui/settings_panel.py` - 设置面板
- `engine/notifier.py` - 通知器（已有 OS 通知能力）
- `data/models.py` - 数据模型（已有 `alert_threshold`、`SiteStatus` 等）

## Goals / Non-Goals

**Goals:**
- 全站中文化，包括所有按钮、标签、提示信息、对话框
- 卡片式仪表盘首页：今日盈利卡片（可点击展开明细）、上游额度概览卡片、告警摘要卡片
- 上游额度低于阈值时整行标红 + 弹窗提醒 + 系统托盘通知
- 高效的站点配置体验：快速添加/编辑/删除上游站点
- 盈利详情展开视图：点击盈利数字展示主站消耗与各上游消耗的分项明细

**Non-Goals:**
- 不做 Web 端或移动端适配
- 不做国际化框架（i18n），直接硬编码中文即可
- 不做历史数据趋势图表（后续迭代）
- 不做自动巡检定时任务（本次仅手动触发）

## Decisions

### 1. 中文本地化方案：直接替换 vs Qt i18n 框架

**选择：直接替换硬编码中文**

理由：项目仅面向中文站长，不需要多语言支持。使用 Qt 的 `QTranslator` 框架会增加不必要的复杂度（.ts 文件管理、编译 .qm 文件等）。直接将所有 UI 字符串改为中文，简单直接。

### 2. 仪表盘布局方案：卡片式 Grid vs 传统线性布局

**选择：卡片式 Grid 布局（QGridLayout + QFrame 卡片）**

理由：卡片式布局符合现代仪表盘设计范式，信息密度高且层次分明。PySide6 的 `QFrame` + `QGraphicsDropShadowEffect` 可实现美观的卡片效果，无需引入额外 UI 库。

布局结构：
```
┌─────────────────────────────────────────┐
│  📊 今日盈利卡片 (可点击展开明细)         │
├──────────────────┬──────────────────────┤
│  上游额度概览     │  告警摘要             │
│  (上游列表卡片)   │  (低于阈值的上游)      │
├──────────────────┴──────────────────────┤
│  详细状态列表 (全部站点)                   │
└─────────────────────────────────────────┘
```

### 3. 阈值告警 UI 方案：仅标红 vs 标红 + 弹窗 + 托盘通知

**选择：三层告警机制**

- **第一层**：列表中低于阈值的行自动标红背景 + 红色文字（被动告警）
- **第二层**：巡检完成后如有告警，弹出 `QMessageBox` 汇总（主动告警）
- **第三层**：通过已有 `notifier.py` 发送系统托盘通知（后台告警）

理由：三层机制确保站长在任何场景下都不会遗漏告警——看屏幕时有视觉标记，巡检完有弹窗，切到其他窗口时有系统通知。

### 4. 盈利详情展开方案：新窗口 vs 就地展开

**选择：就地展开（QAnimationGroup + QFrame）**

理由：就地展开避免弹窗打断操作流，用户可快速浏览后收起。使用 `QPropertyAnimation` 实现平滑展开/收起动画，体验更流畅。

### 5. 样式方案：Qt StyleSheet vs 自定义 QStyle

**选择：Qt StyleSheet（QSS）**

理由：项目已使用内联 StyleSheet，保持一致性。将样式提取到独立的 QSS 常量模块中，便于统一管理颜色、字体、间距等设计令牌。

## Risks / Trade-offs

- **[中文硬编码] → 后续若需英文版需大量修改**：可接受，当前用户群仅为中文站长，YAGNI 原则
- **[卡片式布局增加 UI 复杂度] → 适度封装为可复用的 CardWidget 组件**：降低单个文件的行数
- **[三层告警可能打扰用户] → 弹窗仅在有新告警时出现**：不重复弹窗已确认的告警
- **[就地展开动画在低配机器可能卡顿] → 提供无动画降级**：检测是否支持硬件加速
