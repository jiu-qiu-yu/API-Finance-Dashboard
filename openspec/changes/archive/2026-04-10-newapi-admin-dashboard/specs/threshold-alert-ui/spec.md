## ADDED Requirements

### Requirement: 低于阈值自动标红
巡检完成后，系统 SHALL 自动将余额低于或等于阈值的上游站点行背景设为红色，文字加粗为白色，以醒目方式提示站长。

#### Scenario: 单个上游低于阈值
- **WHEN** 巡检完成，上游 "ProviderA" 余额 $3.00 低于阈值 $5.00
- **THEN** "ProviderA" 所在行背景变为红色，文字为白色加粗

#### Scenario: 余额恢复正常
- **WHEN** 下次巡检时 "ProviderA" 余额 $8.00 高于阈值 $5.00
- **THEN** "ProviderA" 所在行恢复正常背景色

### Requirement: 告警弹窗汇总
巡检完成后，若存在低于阈值的上游站点，系统 SHALL 弹出一个汇总告警弹窗，列出所有告急的上游站点名称、当前余额和阈值。弹窗 SHALL 使用警告图标和红色标题。

#### Scenario: 多个上游告急弹窗
- **WHEN** 巡检完成，"ProviderA"（余额 $3.00 / 阈值 $5.00）和 "ProviderB"（余额 $1.00 / 阈值 $3.00）告急
- **THEN** 弹出告警弹窗，标题为 "额度告急警告"
- **AND** 内容列出："ProviderA: 余额 $3.00（阈值 $5.00）" 和 "ProviderB: 余额 $1.00（阈值 $3.00）"

#### Scenario: 无告急不弹窗
- **WHEN** 巡检完成，所有上游余额均高于阈值
- **THEN** 系统不弹出告警弹窗

### Requirement: 系统托盘通知告警
当存在告急上游时，系统 SHALL 通过已有的通知器发送操作系统级通知，确保站长即使切换到其他窗口也能收到提醒。

#### Scenario: 系统通知发送
- **WHEN** 巡检完成，存在告急上游
- **THEN** 系统发送 OS 通知，标题为 "New API 额度告急"
- **AND** 内容包含告急上游数量和名称摘要
