## Why

当前仪表盘虽然功能完备，但 UI 设计存在几个影响效率的问题：卡片间信息层级不清晰，站长需要反复查看才能定位问题；利润展示与告警信息视觉权重相近，无法做到"一眼看懂今日状况"；额度概览卡片和状态表格展示同一批数据的两种视角，15+ 站点时两个区域都需要滚动，造成认知分裂。作为中转站长的核心工作台，需要让他们在 3 秒内就能判断"今天是否正常、利润多少、哪里有问题"。

## What Changes

- **重构仪表盘布局架构（S3 合并方案）**：将原来的"三卡片 + 状态表格"四区域布局，重构为"利润卡片 + (告警侧栏 | 超级表格)"三区域布局。删除独立的"额度概览"卡片，将进度条融入状态表格，消除双滚动问题
- **升级视觉设计系统**：引入现代化配色方案（窗口 `#f0f2f5` / 卡片 `#fafbfc`）、三级阴影、字重令牌、统一圆角体系
- **优化利润卡片展示**：增加顶部紧凑摘要行（站点数 | 告警数 | 总消耗），添加趋势指示符号（↑/↓），明细面板设置 max-height 内部滚动
- **告警卡片缩小为侧栏**：缩窄为 ~200px 固定宽侧栏，使用浅红背景 + 左侧 4px 红色边条增强紧迫感（适配"偶尔告警"场景）
- **超级表格替代额度概览 + 状态表格**：新增"额度比"列（内嵌迷你进度条），交替行底色，现代化状态徽章，150ms hover 过渡效果
- **优化设置面板**：改进表单布局、输入框焦点状态、按钮层级
- **删除 `quota_list_widget.py` 的独立使用**：其功能被超级表格吸收

## Capabilities

### New Capabilities
- `modern-theme-system`: 现代化设计令牌体系（配色梯度、三级阴影、字重、圆角、间距），替换 styles.py 中的基础常量
- `dashboard-summary-bar`: 利润卡片顶部紧凑摘要行，显示"站点: N | 上游: N | 告警: N | 总消耗: $X"
- `unified-site-table`: 统一站点健康总览表格（替代原 enhanced-status-table），合并额度概览和状态表格功能，含内嵌进度条列、交替行色、状态徽章、hover 效果

### Modified Capabilities
- `dashboard-ui`: 利润展示区域重设计 + 明细面板 max-height 约束 + 布局从四区域变三区域
- `dashboard-cards`: **BREAKING** 布局架构重构 — 从"利润 + (额度3:告警2) + 表格"变为"利润 + (告警侧栏1:表格4)"，删除独立额度概览卡片
- `upstream-quota-monitor`: **BREAKING** 独立卡片形式删除，进度条逻辑迁移至 unified-site-table 的额度比列
- `threshold-alert-ui`: 告警卡片缩小为侧栏，浅红背景 + 左侧红色边条设计

## Impact

- **删除/弃用文件**：`quota_list_widget.py` 的独立卡片用法不再需要（进度条逻辑可迁移至超级表格）
- **大幅改动文件**：`styles.py`、`main_window.py`（布局重构）、`status_list.py`（升级为超级表格）
- **中等改动文件**：`card_widget.py`、`settings_panel.py`、`collapsible_panel.py`
- **无后端变更**：纯前端 UI 层改动，不涉及数据层、引擎层或配置存储
- **无 API 变更**：所有已有的数据接口保持不变
- **依赖**：不引入新的第三方依赖，仅使用 PyQt6 原生样式能力
