## Context

当前 API 财务仪表盘是一个 PyQt6 桌面应用，为中转站长提供每日利润监控、上游额度跟踪和告警功能。典型用户有 **15+ 上游站点**，告警为**偶发事件**（需要醒目提醒）。

**现状布局：** 利润卡片(全宽) → 额度概览(3)+告警摘要(2) → 状态表格(全宽)
**核心问题：** 15+ 站点时，额度概览和状态表格都需要滚动，展示同一批数据的两种视角，造成认知分裂和双滚动体验灾难。

**约束条件：**
- 纯 PyQt6 原生样式（QSS + QGraphicsDropShadowEffect），不引入第三方 UI 框架
- 保持所有现有功能和数据接口不变
- 最小窗口 960×700px，Windows 平台

## Goals / Non-Goals

**Goals:**
- 站长 3 秒内判断"利润多少、是否有问题"（F 型视觉流引导）
- 消除双滚动问题：所有站点数据在单一可滚动的"超级表格"中
- 告警偶发时能立刻引起注意（强告警侧栏设计）
- 建立完整的设计令牌体系（三级阴影、字重、圆角、配色梯度）
- 15+ 行数据表格的高可读性（交替行色、状态徽章、hover 效果）

**Non-Goals:**
- 不做暗黑模式 / 主题切换
- 不引入图表库
- 不做响应式移动端适配
- 不改变数据层 / 引擎层逻辑
- 不做脉冲/呼吸动画，仅使用 hover 过渡效果

## Decisions

### Decision 1: S3 合并方案 — 删除独立额度概览卡片

**选择：** 将"额度概览"和"状态表格"合并为一个"超级表格"，增加"额度比"列（内嵌迷你进度条）。告警卡片缩小为 ~200px 固定宽侧栏。

**新布局：**
```
┌─────────────────────────────────────────────────────┐
│  [利润卡片 + 摘要行]                    (全宽, 固定)  │
├──────────────┬──────────────────────────────────────┤
│  [告警侧栏]   │  [站点健康总览 - 超级表格]             │
│  (~200px)    │  (~712px, 唯一滚动区)                 │
│  (固定高度)   │  含: 站点名|消耗|余额|额度比|状态       │
└──────────────┴──────────────────────────────────────┘
```

**理由：**
- 15+ 站点时，两个独立列表（额度概览 + 状态表格）各自滚动 = 体验灾难
- 合并后只有一个滚动区域，同一行看到站点的全部信息
- 告警通常只有 0-3 个站点，200px 侧栏足够

**空间预算：**
- 利润卡片: ~175px 高（摘要行+数字+公式+按钮+时间）
- 间距: 24px
- 下半区: 700 - 175 - 24 - 32(边距) = ~469px
- 表格可见行: (469 - 40 - 36) ÷ 36 ≈ 10.9 行 → 15+ 行只需滚动 4-5 行

**备选方案：**
- S1 保持现有结构 → 双滚动问题未解决
- S2 整页滚动 → 滚动后利润/告警不可见

**影响：** `quota_list_widget.py` 的独立卡片用法将被弃用，进度条逻辑迁移至超级表格。

### Decision 2: 在 styles.py 中扩展设计令牌体系

**选择：** 扩展现有 `styles.py`，增加 `Shadows`（三级）、`FontWeights`、`BorderRadius` 类

**新增令牌：**
- 阴影: `SM(2px)` / `MD(8px)` / `LG(16px)` — 按钮/卡片/对话框
- 字重: `REGULAR(400)` / `MEDIUM(500)` / `SEMIBOLD(600)` / `BOLD(700)`
- 圆角: `SM(4px)` / `MD(8px)` / `LG(12px)`
- 背景: 窗口 `#f0f2f5` / 卡片 `#fafbfc` / 浅色语义背景

**理由：** 保持单一样式来源，利用现有引用关系，无需引入外部样式系统。

### Decision 3: 配色方案微调而非重做

**选择：**
- 窗口背景: `#f5f6fa` → `#f0f2f5`（更深，增加卡片对比度）
- 卡片背景: `#ffffff` → `#fafbfc`（微灰白，更柔和）
- 语义色保持不变（Success/Danger/Warning）
- 新增浅色语义背景: `#f0faf4`(浅绿) / `#fff2f0`(浅红) / `#fff8e6`(浅橙)

**理由：** 用户已习惯当前颜色语义，只需微调背景层次即可显著提升现代感。

### Decision 4: 紧凑摘要行集成在利润卡片内

**选择：** 利润卡片顶部一行文字："站点: 17 | 上游: 15 | 告警: 2 | 总消耗: $342.50"

**理由：** 避免独立卡片占用空间，摘要与利润信息高度相关，12px 灰色文字不抢注意力。

### Decision 5: 告警侧栏 — 浅红背景 + 左侧红色边条

**选择：** 告警时整个侧栏卡片使用 `#fff2f0` 背景 + 左边框 4px `Colors.DANGER`。无告警时使用正常白色 + 绿色文字。

**理由：** 偶发告警需要强视觉提醒。左侧边条是现代 UI 的通用"注意"pattern（类似 VS Code warning panel）。

### Decision 6: 超级表格 5 列设计 + hover 效果

**选择：** 站点名(stretch) | 今日消耗(100px) | 当前余额(100px) | 额度比(150px, 含进度条) | 状态(90px, 徽章)

**hover 效果：**
- 普通行: 背景变为 `#f5f7fa`，150ms ease
- 告急行: 背景从 `#fff2f0` 变为 `#ffe8e5`，150ms ease

**理由：** 5 列在 712px 宽度内可以放下，hover 效果增加交互反馈感。

### Decision 7: 明细面板 max-height 约束

**选择：** CollapsiblePanel 展开时设置 max-height: 200px，超出部分在面板内滚动。

**理由：** 15+ 站点的消耗明细可能很长，不约束的话会把下方表格推出视口。

### Decision 8: 告警侧栏自然高度，不强制等高

**选择：** 告警侧栏使用自然高度，下方留白而不拉伸到表格等高。

**理由：** 告警内容通常 0-3 行，强制等高会产生大量空白，不如让侧栏紧凑自然。使用 `Qt.AlignmentFlag.AlignTop` 对齐。

## Risks / Trade-offs

- **[布局重构范围大]** S3 方案需要重写 main_window.py 的布局逻辑和 status_list.py → **缓解：** 分阶段实施，先令牌体系 → 利润卡片 → 超级表格 → 告警侧栏 → 设置面板
- **[进度条在表格中的实现]** QTableWidget 嵌入 QProgressBar 需要 setCellWidget → **缓解：** 验证性能，15 行 × 1 个 widget 应在可接受范围
- **[quota_list_widget 弃用]** 需要确保没有其他地方依赖它的独立卡片形式 → **缓解：** grep 确认引用
- **[用户适应成本]** 布局变化较大（告警从右侧独立卡片变为左侧侧栏） → **缓解：** 保持所有交互逻辑不变
- **[告急行 hover 性能]** QSS hover 在 QTableWidget 中可能需要 viewport stylesheet → **缓解：** 测试后选择最佳实现方式
